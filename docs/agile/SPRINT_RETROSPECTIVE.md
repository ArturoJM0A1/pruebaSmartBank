# Sprint Retrospective - SmartBank

## Retrospective Purpose

**Duration**: 1-1.5 hours
**Participants**: Development Team, Scrum Master
**Purpose**: Reflect on the sprint, identify improvements, create action items

---

## Retrospective Formats

### 1. Start/Stop/Continue
- **Start**: What should we begin doing?
- **Stop**: What should we stop doing?
- **Continue**: What's working well and should continue?

### 2. 4Ls
- **Liked**: What did we enjoy?
- **Learned**: What did we learn?
- **Lacked**: What was missing?
- **Longed For**: What did we wish we had?

### 3. Mad/Sad/Glad
- **Mad**: What frustrated us?
- **Sad**: What disappointed us?
- **Glad**: What made us happy?

---

## Sprint 1 Retrospective (Start/Stop/Continue)

### Start (Things to begin doing)
1. Write unit tests before code (TDD)
2. Daily code reviews
3. Update task board daily
4. Share knowledge in daily standup

### Stop (Things to stop doing)
1. Committing without running tests
2. Long coding sessions without breaks
3. Skipping daily standup
4. Over-engineering simple features

### Continue (Things working well)
1. Clear sprint goals
2. Good team communication
3. Regular backlog grooming
4. Code review process

## Facilitation Tips

1. **Create safe space**: Everyone can speak honestly
2. **Focus on process**: Not people
3. **Action items must be specific**: Who, what, when
4. **Limit action items**: 3-5 per sprint
5. **Follow up**: Review action items at start of next retro
